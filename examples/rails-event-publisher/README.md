# Rails Event Publisher — Integration Example

**Status:** Integration example — not a full Rails application.

This directory contains an example Ruby service object showing how a Rails application
could publish customer messages into the Reliability Lab pipeline. It demonstrates
the integration pattern without requiring a full Rails app.

## What This Example Shows

- A Rails service object that publishes events to the FastAPI ingestion endpoint
- Idempotency via `message_id` (UUID generated by the caller)
- Timeout and retry considerations
- How a Rails monolith or microservice could use this pipeline

## How This Maps to the Job Stack

Many backend/platform roles mention **Ruby on Rails** alongside Python, Kubernetes,
and cloud infrastructure. This example shows:

1. **Rails as an event producer** — the Rails app publishes events; the Python pipeline
   handles reliability, persistence, and search indexing.
2. **Polyglot architecture** — Rails (Ruby) + FastAPI (Python) + RabbitMQ (messaging)
   is a realistic production pattern. The Rails app doesn't need to know about
   PostgreSQL, Elasticsearch, or retry logic — it just publishes and moves on.
3. **Idempotency at the boundary** — the `message_id` is generated by the Rails app
   and passed to the pipeline. If the HTTP call times out, the Rails app can safely
   retry with the same `message_id`.

## Usage

```ruby
# In a Rails controller, service, or background job:

result = CustomerMessagePublisher.call(
  customer_id: "cust-001",
  text: "مرحبا بالعالم",
  channel: "web",
  message_id: SecureRandom.uuid  # idempotency key
)

if result.success?
  Rails.logger.info "Published message #{result.message_id}"
else
  Rails.logger.error "Failed to publish: #{result.error}"
end
```

## Honest Positioning

This is an integration example showing the pattern. It is not a production Rails
application. A real integration would add:
- Background job wrapper (Sidekiq, GoodJob) for async publishing
- Circuit breaker (Semian, resilience) for API outages
- Configuration per environment (development/staging/production API URLs)
- API key authentication
- Test suite with WebMock/VCR
